## 1. Property Listings (Guest)

### [v1] Browse available properties
As a guest, I want to see all available properties on the homepage, so that I can quickly understand what's on offer and choose which one to explore.

**Acceptance criteria:**
- Homepage displays a card for each property (3 total)
- Each card shows: property name, location, a lead photo, and a starting nightly rate
- Clicking a card navigates to the full property listing page
- All properties are always shown — no search, filter, or pagination required

Labels: `[v1]`

---

### [v1] View a property listing page
As a guest, I want to view a full property listing page, so that I have enough information to decide whether to book.

**Acceptance criteria:**
- Page displays: photo gallery, property name, location, short description, key details (bedrooms, bathrooms, max guests), amenities list, and nightly rate
- An availability date picker is visible and reflects real-time availability
- A clear call-to-action ("Book now" or "Check availability") is present above the fold
- If selected dates are unavailable, the guest is informed immediately — not after attempting to book

Labels: `[v1]`

---

## 2. Instant Booking & Payment (Guest)

### [v1] View booking summary and price breakdown
As a guest, I want to see a full price breakdown before I pay, so that I know exactly what I'm being charged and why.

**Acceptance criteria:**
- Breakdown displays: nightly rate × number of nights, cleaning fee, refundable deposit, and total amount
- Selected dates and property name are clearly shown
- Deposit is labelled as refundable with a note on when it is returned
- Guest cannot proceed to payment without reviewing this screen

Labels: `[v1]`

---

### [v1] Complete booking via card payment (Stripe)
As a guest, I want to pay by card, so that I can confirm my booking instantly.

**Acceptance criteria:**
- Guest is presented with a Stripe-hosted payment form
- Total charge includes nightly fees, cleaning fee, and deposit as a single transaction
- On successful payment, booking is confirmed immediately and a confirmation is triggered
- On payment failure, the guest is shown a clear error and can retry — dates are not blocked
- Booking is written to the relevant property's Google Calendar upon confirmation

Labels: `[v1]` `[External API]`

---

### [v1] Complete booking via bank transfer (Wise)
As a guest, I want to pay by bank transfer, so that I can book without using a card.

**Acceptance criteria:**
- Guest can select "Pay by bank transfer" as an alternative to card
- Guest is shown the Wise account details (account number, reference to use, amount to transfer)
- Booking is created in a "pending payment" state — dates are soft-blocked but not confirmed
- Admin receives a notification of a pending bank transfer booking
- Admin can manually mark the booking as paid, which triggers confirmation and blocks the Google Calendar
- Pending bookings not marked as paid within a configurable window (e.g. 48h) can be cancelled by admin

Labels: `[v1]`

---

## 3. Booking Confirmation (Guest)

### [v1] Receive booking confirmation email
As a guest, I want to receive a confirmation email immediately after booking, so that I have all the details I need for my stay.

**Acceptance criteria:**
- Email is sent automatically upon successful payment (Stripe) or when admin marks a bank transfer as paid
- Email includes: property name, guest name, check-in and check-out dates, full price breakdown (nightly fees, cleaning fee, deposit), and booking reference number
- Email includes the property's current check-in instructions (pulled from the admin-managed text field at time of booking)
- Email is sent to the address provided during booking
- Email is clearly branded and readable on mobile

Labels: `[v1]`

---

### [v1] Admin updates check-in instructions per property
As an admin, I want to edit the check-in instructions for each property, so that guests always receive accurate arrival information including the current door code.

**Acceptance criteria:**
- Each property has a dedicated check-in instructions text field in the admin panel
- Admin can update the field at any time
- Changes take effect on all confirmation emails sent after the update — previously sent emails are unaffected
- Field supports basic formatting (line breaks at minimum)

Labels: `[v1]`

---

## 4. Admin — Booking Management

### [v1] View all bookings
As an admin, I want to see a list of all bookings across all properties, so that I can manage the schedule and stay on top of upcoming stays.

**Acceptance criteria:**
- Bookings are displayed in a list with: guest name, property, check-in/check-out dates, total amount, payment method, and booking status (confirmed, pending payment, cancelled)
- List can be filtered by property and by status
- Clicking a booking opens its detail view

Labels: `[v1]`

---

### [v1] Create a booking manually
As an admin, I want to create a booking manually, so that I can log stays that were arranged directly with a guest outside the website.

**Acceptance criteria:**
- Admin can create a booking by entering: guest name, email, property, check-in/check-out dates, and total amount
- Admin sets payment status at creation — either "paid" or "unpaid"
- Booking is written to the relevant property's Google Calendar upon creation
- Booking confirmation email is sent to the guest upon creation
- Booking is marked as "manually created" in the booking record

Labels: `[v1]` `[External API]`

---

### [v1] Cancel a booking
As an admin, I want to cancel a booking, so that I can free up availability when a guest cancels or a stay falls through.

**Acceptance criteria:**
- Admin can cancel any booking from the booking detail view
- Cancellation removes the event from the property's Google Calendar
- Booking status is updated to "cancelled"
- Admin is reminded to process any applicable refund manually (via Stripe dashboard or Wise) — no automated refund is triggered
- A cancellation notification email is sent to the guest

Labels: `[v1]` `[External API]`

---

## 5. Admin — Calendar Sync

### [v1] Connect a calendar per property
As an admin, I want to connect a calendar to each property via CalDAV, so that availability on the website reflects all bookings including those from OTA platforms.

**Acceptance criteria:**
- Admin can connect a CalDAV calendar to each property in the admin panel
- Each property has its own calendar connection — three calendars total
- Admin can disconnect and reconnect a calendar without losing existing booking data
- Connection status is visible per property (connected / disconnected)

Labels: `[v1]` `[External API]`

---

### [v1] Sync availability via CalDAV
As an admin, I want the website availability to stay in sync with each property's calendar via CalDAV, so that guests cannot book dates that are already blocked by any source — OTA, manual, or direct bookings.

**Acceptance criteria:**
- Admin connects each property's calendar using CalDAV credentials (one per property)
- System polls each calendar via CalDAV every 15 minutes and updates availability accordingly
- Any event found on the calendar blocks the corresponding dates on the website date picker
- Bookings made on the website immediately create a CalDAV event on the relevant property's calendar upon confirmation
- If a CalDAV connection is interrupted, affected dates are shown as unavailable by default until sync is restored
- Admin is notified if a sync fails

Labels: `[v1]` `[External API]`

---

## 6. Admin — Property & Listing Management

### [v1] Create and publish a property listing
As an admin, I want to create a property listing and publish it to the website, so that guests can find and book the property.

**Acceptance criteria:**
- Admin can create a listing with: property name, location, short description, bedroom count, bathroom count, max guests, amenities list, nightly rate, cleaning fee, and deposit amount
- Admin can upload photos directly to Cloudinary via the admin panel — no manual URL entry required
- Listing has a draft/published toggle — unpublished listings are invisible to guests
- Check-in instructions field is included per property
- CalDAV calendar connection is configured per property from this screen

Labels: `[v1]` `[External API]`

---

### [v1] Edit a property listing
As an admin, I want to edit any property listing at any time, so that I can keep details, pricing, and photos accurate.

**Acceptance criteria:**
- Admin can edit all listing fields at any time — changes go live immediately on save
- Nightly rate, cleaning fee, and deposit can be updated independently
- Photos can be added, removed, or reordered
- Editing a published listing does not unpublish it — changes are reflected immediately

Labels: `[v1]`

## Technical notes

- **Payments:** Stripe for v1, implemented with a clean provider pattern (simple interface that Stripe implements) to make adding future providers straightforward without a full abstraction layer upfront
- **Calendar sync:** Use `tsdav` library for CalDAV integration — this is the same library Cal.com uses under the hood
- **Photo hosting:** Cloudinary for v1 — free tier sufficient at this scale, handles image optimisation and CDN delivery automatically
- **OTA availability:** Airbnb and other OTA bookings flow into Google Calendar via iCal export (already configured) — CalDAV sync picks these up automatically, no direct OTA integration needed in v1
